# Deribit Price Tracker API

Асинхронный сервис на FastAPI для мониторинга курсов криптовалют с биржи Deribit. Проект автоматически собирает данные о ценах, сохраняет их в базу данных и предоставляет API для доступа к истории и актуальным значениям.

## Функционал
* **Фоновый сбор данных**: Интеграция с Deribit API через Celery.
* **Асинхронный API**: Высокая производительность благодаря FastAPI и SQLAlchemy 2.0.
* **Контейнеризация**: Полная настройка Docker для быстрой развертывания.
* **Тестовое покрытие**: Автоматизированные асинхронные тесты.

---

## Стек технологий
* **Backend**: Python 3.12, FastAPI
* **Database**: PostgreSQL (SQLAlchemy 2.0 + Alembic)
* **Task Queue**: Celery + Redis
* **Testing**: Pytest + Pytest-Asyncio + HTTPX
* **Environment**: Docker / Docker Compose

### 2. Запуск проекта

Запустите все сервисы одной командой:

-docker-compose up -d --build


* **API**: http://localhost:8000
* **Документация Swagger**: http://localhost:8000/docs

Миграции: Выполняются автоматически при старте сервиса api.

### 3. Запуск тестов

Тесты запускаются в изолированной среде внутри контейнера:

-docker exec -it deribit_api pytest tests/

## Design Decisions (Архитектурные решения)
В ходе разработки были приняты следующие решения для обеспечения надежности и чистоты кода:

* **Lazy Engine Initialization**: Движок SQLAlchemy инициализируется "лениво". Это позволило избежать классической ошибки attached to a different loop, возникающей при импорте асинхронных драйверов в тестах.

* **NullPool в тестовой среде**: В тестах используется poolclass=NullPool, что гарантирует немедленное закрытие соединений с БД после завершения асинхронного цикла теста, предотвращая зависание процессов.

* **Dependency Overrides**: В тестах используется механизм переопределения зависимостей FastAPI (dependency_overrides). Это позволяет тестам использовать ту же сессию базы данных, что и основному приложению, обеспечивая консистентность данных.

* **Celery Beat Separation**: Планировщик (beat) и исполнитель (worker) вынесены в отдельные контейнеры. Это позволяет масштабировать обработку задач независимо от нагрузки на API.

* **Data Validation**:
Входные параметры (например, тикеры) нормализуются с помощью Pydantic и методов обработки строк, что делает API устойчивым к регистру символов.

### Структура проекта

app/api — Роуты и логика обработки запросов.

app/client — Запрос к deribit.

app/core — Конфигурация проекта (pydantic-settings).

app/db — Модели данных и логика подключения к PostgreSQL.

app/tasks — Задачи для фонового сбора данных через Celery.

tests — Асинхронные тесты API и внешних клиентов.
